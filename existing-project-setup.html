Data Science Repository Setup Generator

Generate a complete Windows setup for a brand-new project, an existing repository with requirements.txt, or an existing notebook repository without dependency files. The commands use Python, uv, Jupyter, VS Code, Git Bash or PowerShell, and a project-local .venv.

The notebook kernel and VS Code interpreter must point to the project environment: .venv\Scripts\python.exe. Do not select the global Python installation under AppData\Local\Programs\Python.
Git Bash example: /e/Data Science
For an already-cloned local repository, leave this empty.

VS Code interpreter and Jupyter server selection

1. Select the Python interpreter

Press Ctrl + Shift + P, run Python: Select Interpreter, then choose .venv\Scripts\python.exe.

2. Select the Jupyter server interpreter

Press Ctrl + Shift + P. If your VS Code Jupyter extension shows Jupyter: Select Interpreter to Start Jupyter Server, choose the same .venv\Scripts\python.exe.

3. Select the notebook kernel

Open the .ipynb file, click Select Kernel, and choose the generated kernel name or the Python environment under .venv.

Fix the “requires the ipykernel package” message

First verify that ipykernel is installed in the project environment:

uv run python -c "import sys, ipykernel; print(sys.executable); print(ipykernel.__version__)"

Correct path: ...\YOUR-REPOSITORY\.venv\Scripts\python.exe

Wrong/global path: C:\Users\...\AppData\Local\Programs\Python\Python312\python.exe

If the project-local import succeeds, do not reinstall ipykernel globally. Change the notebook kernel and VS Code interpreter to the project’s .venv.

The generated commands safely skip uv init --bare when an existing pyproject.toml is already present. The .venv directory is excluded from Git and can be recreated with uv sync.
